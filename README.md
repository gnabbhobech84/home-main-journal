# home-main-journal

一个用于归档和发布多个独立 HTML 页面的仓库。这里存放各类静态网页资源，适合作为个人或小型项目的页面集合。

## 项目简介

本仓库旨在提供一个结构化的空间，用于保存和展示独立的 HTML 文件。每个页面均可独立访问，不依赖特定域名或后端服务。适合存放文档、工具页面、资料页、小型演示等。

## 目录说明

```
home-main-journal/
├── pages/            # 存放所有 HTML 页面文件
├── assets/           # 共用资源（样式、脚本、图片等）
├── index.html        # 入口页面（可选）
└── README.md         # 本文件
```

- `pages/`：每个子文件夹或文件对应一个独立页面，命名清晰直观。
- `assets/`：包含页面间共享的 CSS、JavaScript、字体、图片等资源文件。

## 页面归档说明

- 所有页面均为纯前端 HTML 文件，无需服务器环境即可直接打开。
- 页面间相互独立，不共享状态或数据。
- 每个页面可以附带自己的资源文件夹（如 `page-name/assets/`），也可引用全局资源。

## 维护说明

- 欢迎提交 Pull Request 添加新页面或改进现有页面。
- 新增页面请放入 `pages/` 目录，并尽量保持命名规范。
- 请勿在页面内包含恶意代码、跟踪脚本或违反平台政策的内容。
- 资源文件请尽量压缩，避免仓库体积过大。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意 HTML 文件。
3. 也可部署到任意静态托管服务（如 GitHub Pages、Netlify 等）。
